# KIGTTS 需求说明（原 KGTTS）

> 命名说明：`KIGTTS` 为当前主名称，`KGTTS` 为历史名称，后续逐步弃用。

## 1. 目标体验

### Android 端
1. 导入离线 ASR 模型（`sosv.zip` / `sosv-int8.zip`）。
2. 导入音色包（`voicepack.zip`，由训练器导出）。
3. 开始后持续监听：
   - 说话结束后快速输出识别结果并触发 TTS
   - ASR 与 TTS 并行，不互相阻塞

### PC / Electron 训练端
- 拖入录音后一键完成：预处理 -> 切分 -> 转写 -> 训练/微调 -> 导出 `voicepack.zip`

## 2. 硬约束

- 全链路离线（训练与推理均可离线运行）
- 低延迟（句末触发后尽快开始播报）
- 支持 A/B 音频质量档位
- 模型与音色包可导入、切换、删除
- 连续运行稳定

## 4. 功能模块

### 4.1 训练端（PC/Electron）
- 项目管理（新建/打开/输出目录）
- 音频导入（多格式）
- A/B 档参数控制
- 阶段进度显示（预处理、切分、转写、训练、导出）
- 可选转写抽检与修正

### 4.2 Android 端
- 模型管理：导入/切换/删除 ASR 与音色包
- 语音实时处理：持续监听、句末触发、并行播报
- 运行状态与设备状态可见
- 悬浮窗入口与交互（如启用）

## 5. `voicepack.zip` 最小规范

必须包含：
- `manifest.json`
- `tts/model.onnx`
- `tts/model.onnx.json`
- `tts/phonemizer.dict`

`manifest.json` 关键字段：
- `id` / `name`
- `lang`
- `engine`
- `sample_rate`
- `quant`
- `files`

## 6. 验收标准

### Android
- 断网可运行
- 句末触发播报有效
- 播放时仍可继续识别
- 模型/音色包导入与切换可用

### 训练端
- 一键流程可完成并导出 `voicepack.zip`
- A 档稳定可用，B 档在增强策略下可用

### 端到端
- 训练端导出的 `voicepack.zip` 可在 Android 成功导入并生效
