# 离线实时 ASR → 指定音色 TTS（PC 训练 + Android 端侧）开发说明

## 1) 要做成什么（最终体验）

用户在 Android 手机上打开 App：

1. 选择/加载离线 ASR 模型（sosv-model）。
2. 选择/加载一个“音色包”（voicepack.zip，来自 PC 训练器导出）。
3. 点击开始后：
   * 手机一直听用户说话（ASR 持续运行）
   * 用户每说完一句话，ASR 确认该句结束后，**立刻用指定音色 TTS 播放这一句**
   * **ASR 和 TTS 必须并行**：播放时也继续听，不要互相阻塞

PC 上提供一个训练器（带 GUI）：

* 用户把录音音频拖进去（质量在 A/B 之间可选）
* 自动完成：切分 → 转写 →（可选校对）→ 训练/微调 → 导出 **voicepack.zip**
* voicepack 拷到手机即可用

---

## 2) 必须满足的硬约束

* **全程离线**：手机端 ASR/TTS 不依赖联网
* **实时交互**：一句话结束后尽快开始播报（目标：中端机 < 1.2s 首包，能更快更好）
* **音频质量等级**：支持 A 档（较干净）与 B 档（轻噪声/轻混响）
* **模型与包可导入**：用户可手动导入模型/音色包文件
* **稳定**：连续运行 30 分钟不崩溃、不明显内存泄漏

### 3.2 训练数据素材（用户侧）

* 用户提供的“目标音色”录音（单人声优先）
* 允许 A/B 档：
  * A：较干净录音（推荐优先支持）
  * B：轻噪声/轻混响（需要更强的预处理和筛选）
* 训练器要能处理常见格式：wav/mp3/m4a/flac 等

### 3.3 开发与测试素材（你们准备）

* Android 测试机至少 2 台：中端 + 高端
* 一套固定测试语料（短句/长句/数字/英文夹杂）
* 一段“回声场景”测试（外放播放时继续说话）

## 4) 需要做什么（开发任务按模块分）

## A) PC 端训练器（GUI + 引擎）

### A1. 需要做什么

做一个桌面训练器，用户拖入音频后，一键完成“自动训练”并导出 voicepack。

### A2. 需要做成什么样

**GUI 必须包含：**

* 项目管理：新建项目 / 打开项目 / 选择输出目录
* 导入区：拖拽音频（支持多个文件）
* 参数区：选择 A 档或 B 档（B 档可开关降噪/更严格筛选）
* 进度区：显示每个阶段进度（预处理/切分/转写/训练/导出）
* 片段区（可选但强烈建议）：抽检/校对若干条转写文本（提升最终音色质量）

**引擎必须做：**

1. 音频预处理

* 转成统一格式（单声道、统一采样率）
* A 档：轻量处理（不要过度破坏音色）
* B 档：可选轻降噪 + 更严格丢弃差片段

2. 语音切分（VAD）+ 质量筛选

* 切成 1–12 秒左右的片段（阈值可调）
* 丢弃：太短/太长/能量太低/削波明显/噪声太大片段

3. 自动转写（ASR）生成训练文本

* 生成每个片段的文本与置信度（或等价指标）
* 结果必须能给 GUI 展示与校对

4. 训练/微调 TTS 并导出

* 训练得到“专用音色 TTS 模型”
* 导出一个手机端能加载的格式（建议 ONNX）

5. 打包导出 voicepack.zip

* 导出一个 zip 包，手机导入即可用
* zip 内必须有 manifest + 模型文件 + 必要配置 + phonemizer 资源

### A3. PC 端需要哪些产物（输出）

* `voicepack.zip`（手机端可直接导入）
* 训练日志（方便排错）
* （可选）preview.wav：导出时自动生成一段固定文本的试听音频

---

## B) Android App（离线实时 ASR + 并行 TTS）

### B1. 需要做什么

做一个 Android App，支持离线导入 ASR 模型与音色包，并实时把用户语音识别成文字后用指定音色播报。

### B2. 需要做成什么样

**App 必须包含以下页面/功能：**

1. 模型管理

* 导入 ASR 模型包：`sosv.zip` 或 `sosv-int8.zip`
* 导入音色包：`voicepack.zip`
* 模型与音色包可切换、可删除
* 必须有“模型状态/可用性提示”

2. 实时转换页

* 一个“开始/停止”按钮
* 显示识别结果（至少显示 Final，partial 可选）
* 播放当前输出音频

**并行行为（必须实现）：**

* ASR 一直在跑（持续听）
* 每当 ASR 判断“一句话结束”（endpoint 确认）：
  * 立即把该句文本发给 TTS
  * TTS 立即开始合成并播放
* 播放时也继续听，不要让 TTS 阻塞 ASR

### B3. 回声/复读控制（必须做最低限度）

因为边播边听会把外放声音收进麦克风导致“复读”：

V1 必做最低限度：

* UI 提示：建议使用耳机体验最佳
* 播放期间的“门控策略”（例如：播放时提高 endpoint 阈值/短时间忽略 endpoint/或降低麦克风灵敏触发）

（可选增强）

* 设备支持时启用 AEC（回声消除）能力（best-effort）

### B4. Android 端需要哪些产物（输出）

* 可安装 APK / 可运行工程
* 运行时导入：
  * `sosv.zip` 或 `sosv-int8.zip`
  * `voicepack.zip`

---

## 5) “包”需要做成什么样（只规定必须字段，别过度设计）

## 5.1 voicepack.zip 最小规范（必须）

voicepack.zip 内至少包含：

* `manifest.json`（描述这个音色包：id/name/lang/sample\_rate/engine/quant/files）
* `tts/model.onnx`（或等价推理文件）
* `tts/model.onnx.json`（或等价配置文件）
* `tts/phonemizer.dict`（中文前端需要的最小字典/映射资源）

manifest.json 需要包含这些关键字段：

* id、name
* lang（至少 zh）
* engine（例如 piper-onnx）
* sample\_rate
* quant（fp32/int8 等）
* files（model/config/phonemizer 的相对路径）

## 6) 验收标准（CodeX 完成后怎么判定“做完了”）

### 6.1 Android 端验收

* ✅ 离线：断网也能识别 + 播放
* ✅ 一句话结束触发：说完一句后能自动播报
* ✅ 并行：播放时继续识别，不会“播报中断识别”或“识别中断播报”
* ✅ 可导入与切换：能导入多个 voicepack 并切换音色
* ✅ 连续运行：30 分钟不崩溃

### 6.2 PC 端验收

* ✅ 拖入音频 → 一键流水线跑完 → 导出 voicepack.zip
* ✅ A 档音频可稳定产出可用 voicepack
* ✅ B 档音频在开启增强时能产出更稳的训练集（至少不明显劣化/不崩）

### 6.3 端到端验收

* ✅ PC 导出的 voicepack.zip 能在 Android 上导入并生效（音色变化明显可辨）

---

## 7) CodeX 开工时的默认选择（避免反复摇摆）

为了先跑通闭环，建议 CodeX 默认这样做（后续可优化）：

* Android 端先用 **onnxruntime** 跑 TTS（集成简单）
* 语音包先只做 **FP32**（量化 INT8 放到 V1.1）
* phonemizer 先做最小可用（字/词 → 拼音），缺失回退逐字
* 播放策略默认 **barge-in**（用户开始说话时停止/降低播报），减少回声导致复读
